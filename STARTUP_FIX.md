# Виправлення проблеми зі стартом ботів

## Проблема 1: Боти не відповідали після запуску
Після запуску `run_all.py` боти не відповідали, але після зупинки системи (Ctrl+C) всі повідомлення з'являлися одночасно.

### Причина
Flask сервер не встигав повністю запуститися до того, як боти та ngrok починали надсилати запити. Запити накопичувалися в черзі та оброблялися тільки при зупинці системи.

### Виправлення
1. Додано функцію `wait_for_flask()` в `run_all.py` що перевіряє готовність Flask
2. Додано затримки 2 секунди між запуском кожного бота
3. Додано логування всіх API запитів в `app.py`

## Проблема 2: Величезна кількість з'єднань (~300/хвилину)
В логах ngrok було ~200+ з'єднань за хвилину, всі від одного IP.

### Причина
В `remote_control.html` на лінії 324 було `setInterval(updateScreen, 200)` - це оновлювало скріншот екрану **кожні 200 мілісекунд** (5 разів на секунду, 300 разів на хвилину)!

```javascript
// БУЛО (дуже агресивно):
streamInterval = setInterval(updateScreen, 200); // 5 FPS = 300 запитів/хвилину

// СТАЛО (розумно):
streamInterval = setInterval(updateScreen, 2000); // 0.5 FPS = 30 запитів/хвилину
```

### Виправлення
Змінено інтервал оновлення екрану з 200ms на 2000ms (з 5 FPS на 0.5 FPS). Це зменшує навантаження в **10 разів**.

## Додаткові налаштування

### Якщо потрібно швидше оновлення екрану:
Відредагувати `remote_control.html`, лінія 326:
```javascript
// Для 1 FPS (60 запитів/хвилину):
streamInterval = setInterval(updateScreen, 1000);

// Для 2 FPS (120 запитів/хвилину):
streamInterval = setInterval(updateScreen, 500);
```

⚠️ **Увага:** Чим менший інтервал, тим більше навантаження на сервер та трафік!

### Якщо rate limit спрацьовує:
Відредагувати `app.py`, лінії 40-41:
```python
RATE_LIMIT_WINDOW = 60  # секунд
RATE_LIMIT_MAX_REQUESTS = 30  # максимум запитів за вікно
```

Це може бути сканер або бот. Rate limiting автоматично заблокує після 30 запитів за хвилину.
