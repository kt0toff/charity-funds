# Charity Funds Directory 🤝

Веб-сайт для управління інформацією про благодійні фонди та ресурси, які вони надають.

## 🌟 Особливості

- 📋 Список ресурсів з фільтрацією та сортуванням
- 🎯 Картки фондів з детальною інформацією
- ✏️ Повне редагування (додавання, зміна, видалення)
- 💾 Автоматичне збереження в Netlify Blobs
- 📱 Адаптивний дизайн
- 🌐 Безкоштовний хостинг на Netlify

## 🚀 Деплой на Netlify

### Простий деплой (3 хвилини)

1. Зареєструйтесь на https://netlify.com через GitHub
2. Натисніть "Add new site" → "Import an existing project"
3. Виберіть "Deploy with GitHub"
4. Знайдіть репозиторій `charity-funds`
5. Натисніть "Deploy site"
6. **Готово!** 🎉

Сайт буде доступний за адресою `your-site.netlify.app`

**Ніяких додаткових налаштувань не потрібно!** Netlify Blobs працює автоматично.

## 📝 Як користуватись

1. Відкрийте сайт
2. Натисніть "➕ Додати ресурс" щоб додати категорію (наприклад "Продукти харчування")
3. Оберіть ресурс зі списку
4. Натисніть "➕ Додати фонд" щоб додати благодійний фонд
5. Всі зміни автоматично зберігаються в Netlify Blobs
6. Дані доступні миттєво (без перезавантаження сайту)

## 🎨 Можливості

- ✅ Додавання/редагування/видалення ресурсів
- ✅ Додавання/редагування/видалення фондів
- ✅ Drag & drop для зміни порядку ресурсів
- ✅ Сортування (своє налаштування, А→Я, Я→А, від новіших, від старіших)
- ✅ Експорт даних в JSON
- ✅ Автоматичне збереження в хмарі
- ✅ Адаптивний дизайн (працює на телефоні)

## 🔒 Безпека

- Всі зміни зберігаються в Netlify Blobs
- Дані доступні тільки через ваш сайт
- Автоматичне резервне копіювання від Netlify

## 💰 Вартість

**ПОВНІСТЮ БЕЗКОШТОВНО:**
- Netlify: безкоштовний план

## 🆘 Troubleshooting

### Дані не зберігаються
- Перевірте консоль браузера (F12) на помилки
- Перевірте, чи сайт задеплоєний на Netlify (не локально)
- Netlify Blobs працює тільки на production

### Дані пропали після редеплою
- Netlify Blobs зберігає дані окремо від коду
- Дані не пропадають при редеплої
- Якщо дані пропали, використайте кнопку "💾 Експорт" для резервного копіювання

## 🆘 Вирішення проблем

### Сервер не запускається
```bash
# Встановіть залежності
pip install flask flask-cors
```

### Порт зайнятий
- Windows: Закрийте інші програми або перезавантажте комп'ютер
- Linux: `fuser -k 5001/tcp`

### Дані не зберігаються
- Перевірте що файл `charity_funds.db` не тільки для читання
- Запустіть від імені адміністратора (Windows)

- Для публічного доступу використовуйте ngrok
