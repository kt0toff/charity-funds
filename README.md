# Charity Funds Directory 🤝

Веб-сайт для управління інформацією про благодійні фонди та ресурси, які вони надають.

## 🌟 Особливості

- 📋 Список ресурсів з фільтрацією та сортуванням
- 🎯 Картки фондів з детальною інформацією
- ✏️ Повне редагування (додавання, зміна, видалення)
- 💾 Автоматичне збереження в SQLite базі даних
- 📱 Адаптивний дизайн
- 🌐 Онлайн хостинг на Render.com

### 💻 Локальна версія

#### Windows

1. Встановіть [Python 3.8+](https://www.python.org/downloads/) (поставте галочку "Add Python to PATH")
2. Завантажте проєкт (зелена кнопка "Code" → "Download ZIP")
3. Розпакуйте в зручну папку
4. Подвійний клік на `start_charity_windows.bat`
5. Відкрийте браузер: http://localhost:5001

#### Linux

```bash
# Встановлення залежностей
pip install -r requirements.txt

# Локальний запуск
./start_charity.sh

# З публічним доступом (ngrok)
./start_charity_public.sh
```

## 💾 База даних

Всі дані зберігаються в файлі `charity_funds.db`:
- ✅ Не втрачаються при перезапуску
- ✅ Можна копіювати між комп'ютерами
- ✅ Працює на Windows/Linux/Mac

**ВАЖЛИВО:** При оновленні проєкту НЕ видаляйте файл `charity_funds.db` - там всі ваші дані!

## 🔄 Як отримувати оновлення

### Варіант 1: Завантаження ZIP (найпростіше)
1. Зайдіть на GitHub сторінку проєкту
2. Натисніть зелену кнопку "Code" → "Download ZIP"
3. Розпакуйте в НОВУ папку
4. **Скопіюйте** файл `charity_funds.db` зі старої папки в нову

cd charity-funds

## 🆘 Вирішення проблем

### Сервер не запускається
```bash
# Встановіть залежності
pip install flask flask-cors
```

### Порт зайнятий
- Windows: Закрийте інші програми або перезавантажте комп'ютер
- Linux: `fuser -k 5001/tcp`

### Дані не зберігаються
- Перевірте що файл `charity_funds.db` не тільки для читання
- Запустіть від імені адміністратора (Windows)
