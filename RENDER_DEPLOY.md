- ✅ Безкоштовний хостинг
- ✅ Постійне посилання (наприклад: `charity-funds.onrender.com`)
- ✅ Автоматичні оновлення з GitHub
- ✅ HTTPS (безпечне з'єднання)
- ✅ Доступ з будь-якого пристрою

## 🚀 Крок 1: Завантажте код на GitHub

```bash

git push origin main
```

---

## 🌐 Крок 2: Створіть акаунт на Render.com

1. Зайдіть на https://render.com
2. Натисніть **"Get Started"**
3. Увійдіть через GitHub (рекомендую)
4. Дозвольте доступ до репозиторіїв

---

## 📦 Крок 3: Створіть Web Service

1. На дашборді Render натисніть **"New +"** → **"Web Service"**

2. Підключіть GitHub репозиторій:
   - Знайдіть `charity-funds`
   - Натисніть **"Connect"**

3. Налаштування:
   ```
   Name: charity-funds
   Region: Frankfurt (EU Central) - найближче до України
   Branch: main
   Root Directory: (залишити порожнім)
   Runtime: Python 3
   Build Command: pip install -r requirements.txt
   Start Command: gunicorn charity_server:app
   ```

4. План:
   - Виберіть **"Free"** (безкоштовний)
   - ⚠️ Обмеження: засинає після 15 хв неактивності, перший запит може бути повільним

5. Натисніть **"Create Web Service"**

---

## ⏳ Крок 4: Дочекайтеся деплою

- Render автоматично встановить залежності
- Запустить сервер
- Це займе 2-5 хвилин
- Статус можна бачити в логах

## 💾 База даних на Render

### ⚠️ ВАЖЛИВО: Обмеження безкоштовного плану

На безкоштовному плані:
- База даних зберігається на диску сервера
- **При кожному новому деплої база СКИДАЄТЬСЯ**
- Дані втрачаються при перезапуску

### ✅ Рішення: Persistent Disk (платно $1/міс)

Або використовуйте зовнішню базу даних:

#### Варіант 1: Render PostgreSQL (рекомендую)
1. На Render створіть **"New +" → "PostgreSQL"**
2. Free план (обмеження: 90 днів, потім треба пересоздати)
3. Підключіть до Web Service

#### Варіант 2: Supabase (безкоштовно)
1. Зареєструйтеся на https://supabase.com
2. Створіть проєкт
3. Отримайте PostgreSQL URL
4. Додайте в Environment Variables на Render

---

## 🔧 Налаштування Environment Variables (якщо потрібно)

На Render в налаштуваннях Web Service:
1. Перейдіть в **"Environment"**
2. Додайте змінні:
   ```
   DATABASE_URL=postgresql://...  (якщо використовуєте PostgreSQL)
   ```

## 📊 Моніторинг

На дашборді Render ви бачите:
- ✅ Статус сервера (Running/Sleeping)
- 📈 Логи в реальному часі
- 🔄 Історію деплоїв
- 📊 Використання ресурсів

---

## ⚠️ Обмеження безкоштовного плану

- 🕐 Засинає після 15 хв неактивності
- 🐌 Перший запит після сну: 30-60 сек
- 💾 База даних не зберігається між деплоями
- 🔄 750 годин/місяць (достатньо для одного сервісу)

### Як уникнути засинання:

**Варіант 1: Cron Job (безкоштовно)**
Створіть на Render **"Cron Job"**:
```
Schedule: */14 * * * * (кожні 14 хвилин)
Command: curl https://charity-funds.onrender.com/api/resources
```

**Варіант 2: UptimeRobot (безкоштовно)**
1. Зареєструйтеся на https://uptimerobot.com
2. Додайте монітор для вашого URL
3. Інтервал: 5 хвилин

## 🆘 Вирішення проблем

### Сервер не запускається
1. Перевірте логи на Render
2. Переконайтеся що `requirements.txt` правильний
3. Перевірте `Procfile`

### База даних порожня після деплою
- Це нормально для безкоштовного плану
- Використовуйте Persistent Disk або зовнішню БД

### Сайт повільно відкривається
- Це нормально після сну (15 хв неактивності)
- Налаштуйте Cron Job або UptimeRobot
